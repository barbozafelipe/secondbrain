## ⚖️ Decisões tomadas

| Tópico | Decisão |
|---|---|
| **Domínios e Certificados** | Centralizar todos os domínios externos no Route53 (AWS) para que o EKS gere os certificados automaticamente via cert-manager, evitando gestão manual ou certificados autoassinados internos. |
| **Cluster Elastic (Afinz)** | O Elastic não ficará no mesmo cluster de produção das aplicações; será criado um cluster separado devido à alta demanda de storage/CPU. |
| **Pipeline de Secrets** | A automação de sincronização de secrets terá o tempo de verificação ajustado (de 5 min para 1h ou mais) para economizar custos de API, e as secrets antigas não utilizadas serão apagadas. |
| **Dívida Técnica (Lambdas)** | O time de Cloud não reescreverá o código dos lambdas Node 14 manualmente; a ferramenta "AWS Transformation" gerará os PRs e o time de desenvolvimento deverá assumir a responsabilidade pela validação e deploy. |

## 📖 Glossário Técnico

| Termo | Significado no contexto |
|---|---|
| **Imperva** | WAF / Firewall de aplicação usado atualmente para proteção e roteamento de DNS externo (direct/relay). |
| **Route53 / R53** | Serviço de DNS da AWS onde os domínios da empresa serão centralizados. |
| **AWS Transformation** | Ferramenta assistente da AWS (provavelmente Amazon Q Developer Agent) que lê repositórios legados (ex: Node 14) e propõe o código migrado para versões mais recentes. |
| **BigQuery Reservation** | Modelo de faturamento por capacidade alocada (slots) no BigQuery, que precisa ser calibrado para evitar o "efeito chicote" de custos. |
| **NAT Gateway** | Serviço AWS que permite recursos em sub-redes privadas acessarem a internet. Estava duplicado na Zapay e foi otimizado. |
| **Efeito Chicote (Custos)** | Quando reduz-se o limite de um serviço (ex: BQ Reservation) e o custo de outro componente atrelado a ele (ex: consumo on-demand) dispara compensando a economia. |
