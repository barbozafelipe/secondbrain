### Incidente de Segurança — DNS Bucket (Sem Parar) — Wellington
> Entrada DNS antiga (`img.semparar.net.br`) apontava para um bucket S3 inexistente na região São Paulo.

- Alguém fez varredura de DNS, encontrou a entrada órfã e **criou um bucket com o mesmo nome**
- Resultado: domínio da Sem Parar resolvia para conteúdo de terceiro (imagem de uma pessoa)
- Wellington abriu chamado na AWS com comprovação via registro.br
- **AWS removeu o bucket no sábado** (menos de 24h após o chamado na sexta à noite)
- Entrada estava no DNS desde **2021**

> [!warning] Lição
> Entradas DNS legadas (domínios antigos como `semparar.net.br`) são vetor real de ataque. Quem controla o nome DNS pode fazer bucket hijacking se não houver bucket correspondente.

## 📌 Glossário técnico desta reunião

| Termo | O que é |
|---|---|
| **Karpenter** | Auto-provisionador de nodes no Kubernetes — mais eficiente que Cluster Autoscaler |
| **Spot VMs** | VMs com desconto que podem ser interrompidas pela cloud — ótimas para workloads tolerantes |
| **NodeSelector** | Configuração no Kubernetes para fixar pods em tipos específicos de node |
| **Imperva** | WAF/proxy usado pela Sem Parar para tráfego externo |
| **F5** | Load balancer de rede usado internamente |
| **Cloud Identity** | Serviço do Google para gerenciar usuários e domínios (equivalente ao Azure AD para GCP) |
| **IAP (Identity-Aware Proxy)** | Serviço Google para acesso seguro a recursos sem VPN |
