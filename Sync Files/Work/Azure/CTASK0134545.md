# CTASK0134545 — Inserir Variáveis de Ambiente na Function App Copilot (PRD)

---

## 📋 Descrição Original

> O recurso que deve receber as novas variáveis é: https://portal.azure.com/#@fleetcorbr.onmicrosoft.com/resource/subscriptions/b1ba0975-b0e9-4aa4-838a-cdb4b0c9e058/resourceGroups/stp-dig-rg-copilot-prd/providers/Microsoft.Web/sites/stp-dig-func-copilot-prd/appServices
> 
> As variáveis abaixo devem ser incluída em nosso recurso.
> 
> VARIAVEIS:
> 
> `URA_USO_VEICULO_ENDPOINT`: `perfilUsoVeiculo`
> `URA_ABASTECE_ENDPOINT`: `limitesAbastece`
> `URA_DRIVETHRU_ENDPOINT`: `limiteDriveThru`
> 
> Caso necessário, entrar em contato com a equipe de desenvolvimento do produto para editar as variáveis de ambiente manualmente no recurso.

## 🧠 Briefing da Tarefa

**O que precisa ser feito:** Adicionar **3 variáveis de ambiente** na Function App `stp-dig-func-copilot-prd` na Azure.

**Onde vou atuar:** Portal Azure → Subscription DIGITAL-PROD → RG `stp-dig-rg-copilot-prd` → Function App `stp-dig-func-copilot-prd` → Configuration → Application Settings.

**Principal cuidado:** São 3 variáveis e os valores parecem ser **nomes de endpoints/rotas** (não URLs completas). Confirmar que o time de dev espera exatamente esses valores — sem barra, sem prefixo.

**Como validar no final:** Confirmar que as 3 variáveis aparecem em Application Settings com os valores corretos e que a Function App reiniciou sem erros.

- Os valores parecem ser **nomes de rotas** (camelCase, sem `/` no início). Confirmar que não deveriam ter prefixo como `/api/` ou URL base.

- **Restart automático** — ao salvar Application Settings, a Function App reinicia. Considerar janela de menor impacto se necessário.
