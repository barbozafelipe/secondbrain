5. [ ] ==Como será feita a associação na AWS==

Depois que o grupo aparecer no IAM Identity Center (em groups, seleciona ele e vai em aws accounts), você deverá associar o mesmo grupo às três contas AWS do projeto:

```text
Sem Parar Doc - IA - DEV
Sem Parar Doc - IA - QA
Sem Parar Doc - IA - PROD
```

A associação será feita usando a lógica:

```text
Grupo + Conta AWS + Permission Set
```

Exemplo:

```text
BR_AWSZPY_IA_DEVELOPER + Conta DEV  + Permission Set DEV (BR_PS_IA_DEV) - copiado de BR_PS_CPP_DEV.
BR_AWSZPY_IA_DEVELOPER + Conta QA   + Permission Set QA (BR_PS_IA_QA) - copiado de BR_PS_CPP_QA.
BR_AWSZPY_IA_DEVELOPER + Conta PROD + Permission Set PROD (BR_PS_IA_PROD) - read only all
```

---

 6. [ ] ==Diferença de permissão por ambiente==

Mesmo sendo apenas um grupo no SailPoint, os acessos aos ambientes não precisam ter a mesma permissão.

A diferença será controlada pelos **Permission Sets** na AWS.

Exemplo:

```text
DEV:
Permissões mais amplas para desenvolvimento e ajustes.

QA:
Permissões semelhantes ou próximas às de DEV, conforme necessidade.

PROD:
Permissão mais restrita, preferencialmente somente leitura.
```

Ou seja:

```text
O grupo é o mesmo.
A conta muda.
O Permission Set muda.
```

9. [ ] ==Passo a passo depois que o grupo aparecer na AWS==

Quando o grupo `BR_AWSZPY_IA_DEVELOPER` aparecer no IAM Identity Center:

```text
1. Acessar o AWS IAM Identity Center.
   
2. Criar os permition sets: 
   Permission Set DEV (BR_PS_IA_DEV) - copiado de BR_PS_CPP_DEV.
   BR_AWSZPY_IA_DEVELOPER + Conta QA   + Permission Set QA (BR_PS_IA_QA) - copiado de BR_PS_CPP_QA.
   BR_AWSZPY_IA_DEVELOPER + Conta PROD + Permission Set PROD (BR_PS_IA_PROD) - read only all

3. Ir em Groups.

4. Procurar o grupo:
   BR_AWSZPY_IA_DEVELOPER

5. Selecionar o grupo.

6. Ir em AWS accounts.

7. Associar o grupo às três contas:
   - Sem Parar Doc - IA - DEV
   - Sem Parar Doc - IA - QA
   - Sem Parar Doc - IA - PROD

7. Para cada conta, selecionar o Permission Set correspondente:
   - DEV → Permission Set de DEV
   - QA → Permission Set de QA
   - PROD → Permission Set de PROD/read-only

8. Confirmar as atribuições.

9. Validar se o acesso aparece no portal AWS SSO após login/logoff.
```
