# TASK1247656 — Criação de Usage Plan e API Key (Integração Meta / Agente WhatsApp)

---

## 📋 Descrição Original

> Criação de um plano de uso e API key em produção, para uma nova integração com a Meta (Agente WhatsApp):
> - Através da console da AWS STP PRD, acessar o serviço de API Gateway, menu "Usage Plans" e criar o seguinte plano:
>    - name: plan_Meta_AI
>    - rate: 100
>    - burst: 200
>    - quota: 10000/dia
> - Na aba "Associated Stages", adicionar os seguintes estágios:
>    - MotorCredito v1
>    - AtendimentoWhatsAppHub v1
>    - AtendimentoWhatsAppHubInternal v1
>    - PIXFreeflow v1
>    - PIX v2
> - Na aba "Associated API Keys", adicionar uma nova chave, com nome: key_Meta_AI_WPP e descrição: Chave de acesso para as APIs da integração com o agente da Meta para o WhatsApp, onde as chamadas parte do ambiente on premise da Sem Parar Corpay
> 
> Após a configuração, compartilhar com o requisitante o valor da chave, pois será configurada em uma secret no openshift.

## 🧠 Entendendo a Task

> *Antes de executar, entenda. O cérebro retém melhor quando conecta o novo ao que já conhece.*

### Por que essa task existe?

O time está criando uma nova integração com a **Meta** para um Agente de WhatsApp. Esse agente (que roda on-premise) precisa consumir várias APIs internas na AWS (como Motor de Crédito e PIX). Para que essa comunicação aconteça de forma segura e controlada, precisamos de uma **chave de acesso (API Key)** atrelada a limites de uso específicos (**Usage Plan**), evitando que o agente sobrecarregue nossas APIs.

### O que você vai configurar, exatamente?

Você vai trabalhar no **API Gateway** na conta **AWS STP PRD**. O fluxo é: **Criar Plano -> Associar APIs -> Criar Chave -> Compartilhar**.

| Configuração | Valores a preencher |
|---|---|
| **Usage Plan** | `plan_Meta_AI` <br> Rate: 100 / Burst: 200 / Quota: 10000 por dia |
| **Stages (APIs liberadas)** | 1. MotorCredito v1 <br> 2. AtendimentoWhatsAppHub v1 <br> 3. AtendimentoWhatsAppHubInternal v1 <br> 4. PIXFreeflow v1 <br> 5. PIX v2 |
| **API Key** | Nome: `key_Meta_AI_WPP` <br> (Incluir a descrição informada no chamado) |

> 💡 **Âncora mental:** O Usage Plan é o "contrato" (limites), e a API Key é a "assinatura" (acesso). Você cria o contrato, diz onde ele vale (Associated Stages) e emite a assinatura atrelada a ele. E não esqueça do *last mile*: o valor gerado precisa chegar nas mãos de quem pediu (via meio seguro) para que seja injetado no OpenShift.

- [ ] Salvei o valor da chave (string) gerado para enviar ao requisitante?
- [ ] Compartilhei a chave de forma segura com o Marcio Luis Velasco Guidolin?
