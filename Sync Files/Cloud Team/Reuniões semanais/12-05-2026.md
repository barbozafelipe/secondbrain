### Elastic — Migração para RKE (Wellington + Thiago)

> Decisão confirmada: **não reaproveitar** o Elastic do OpenShift. Subir novo no RKE.

**Motivo**: licenciamento do OpenShift cobra por CPU. Se o Elastic crescer ali dentro, o custo de licença explode. Melhor migrar agora com apenas 1 app + 1 log streaming.

**Arquitetura planejada**:
- Deploy novo no **RKE** (cluster de prod e non-prod)
- **Lifecycle policy** para retenção inteligente:
  - Hot (últimos 7-30 dias): dentro do Elastic
  - Cold (dados antigos): enviar para **S3** via lifecycle
  - Consulta de dados antigos: mais cara, mas rara
- Conta de visualização para time de negócio/BI (read-only)
- Aumento de retenção sem aumento proporcional de custo

**Status**: Wellington ainda fazendo os designs. Prazo final: até o final do ano. Não vai começar agora — quer terminar Afinz primeiro.

## 📌 Glossário técnico desta reunião

| Termo | O que é |
|---|---|
| **CVE** | Common Vulnerabilities and Exposures — identificador de vulnerabilidade de segurança pública |
| **Imagem customizada (EKS)** | AMI do EC2 modificada pelo time para hardening — precisa de rebuild manual para aplicar patches |
| **Sapi** | Sistema legado da Gringo — workloads já descomissionados, banco ainda estava ativo |
| **Cloud Ace** | Consultoria parceira do ecossistema Google — alocada para apoiar a conta Gringo |
| **Baseline slots (BQ)** | Capacidade fixa contratada de processamento no BigQuery — gera custo mesmo sem uso |
| **Auto-scaling slots (BQ)** | BigQuery escala slots sob demanda — paga só quando usa |
| **Lifecycle policy (Elastic)** | Regra que move dados antigos para storage mais barato automaticamente |
| **Hot / Cold (Elastic)** | Tiers de armazenamento: hot = rápido e caro, cold = lento e barato |
| **Transit Gateway** | Serviço AWS que interconecta múltiplas VPCs — essencial para Afinz falar com conta Fintech |
| **Analyze (Dynatrace)** | Módulo de análise de logs do Dynatrace — consome muito com volume alto de logs |
| **Caio Cruvinel** | Novo Account Manager do Google para a conta Gringo |
| **Caio Proviel** | Contato técnico na Gringo — confirmou descomissionamento do Sapi DB |
| **Clayton** | Contato de gestão/cronograma na Afinz/Carvalt |
| **Aparecida (Cida)** | Contato de gestão na Afinz |
| **Hélio** | Gringo — solicitou desligamento do Analyze em staging |
