## 📌 Glossário técnico desta reunião

| Termo | O que é |
|---|---|
| **IPC** | Sigla usada para identificar os ambientes de infra do projeto Afinz (Dev, HML, Prod) |
| **Veleiro** | Ferramenta de GitOps (provavelmente Argo Rollouts ou FluxCD) instalada nos clusters |
| **Otanos** | Versões antigas de ferramentas no cluster que precisam de atualização |
| **SP (banco)** | Banco SQL Server (stored procedures?) no ambiente Dville que ficou fora do upgrade do final de semana |
| **Dville** | Ambiente de desenvolvimento/HML onde um dos bancos roda |
| **Hito/Shisla** | Serviço que estava consumindo muita memória — precisou de instância superior para mitigar |
| **Management Group (Azure)** | Hierarquia de governança acima das subscriptions — policies aplicadas aqui herdam para tudo abaixo |
| **MCP** | Model Context Protocol — padrão para conectar agentes de IA a ferramentas/sistemas externos |
| **HPA** | Horizontal Pod Autoscaler — escala pods Kubernetes baseado em métricas (CPU, requisições, etc.) |
| **Prometheus** | Ferramenta open-source de coleta de métricas — usado como backend para HPA customizado |
| **GKE Autopilot** | Modo gerenciado do GKE onde o Google provisiona nodes e alguns serviços automaticamente |
| **Cloud Health** | Ferramenta de FinOps para análise e exportação de custos de cloud |
| **Compass** | Provedor/parceiro que emite nota de consumo e suporte da AWS mensalmente |
| **Desmembramento** | Processo de distribuir o custo total da AWS entre centros de custo das áreas internas |
| **Node 14 (Lambda)** | Runtime Node.js 14 — sem suporte, não aceita criação de novas funções Lambda desde 2027 |
| **Oge** | Ferramenta/agente instalado em máquinas para uso interno (contexto de acesso ao cluster GKE) |
| **Joson** | Pessoa externa que fez solicitação de headers no API Gateway |
