## 📌 Glossário técnico desta reunião

| Termo | O que é |
|---|---|
| **Delinea** | Cofre de credenciais (PAM — Privileged Access Management) alugado pelo time de SI e disponibilizado para o squad de Cloud. Substitui anotações individuais e Secret Manager como fonte de credenciais operacionais. |
| **PamFlow** | Nome do namespace segregado que será criado no GKE Autopilot para isolar o serviço de gateway/tokenização da Zapay (contexto PCI). |
| **PCI Proxy** | Serviço de terceiro contratado pela Zapay para fazer tokenização e destokenização de dados de cartão. Diferente do Sem Parar, que faz internamente. |
| **Security Command Center** | Serviço do Google Cloud para monitoramento de segurança, compliance e auditoria de atividades dentro de projetos GCP. Wellington citou como ferramenta de log de auditoria do time. |
| **Chronicle** | Plataforma de SIEM do Google. Mencionado que existia contrato mas que o time de SI não utilizava. Cloud não assume responsabilidade sobre essa ferramenta. |
| **DataGuard** | Tecnologia Oracle para replicação de banco de dados entre data centers (DC1 ↔ DC2). Parou de funcionar após falha da controladora no DC1. |
| **File Storage (OCI)** | Serviço de armazenamento NFS gerenciado da Oracle Cloud. Permite montar um volume remoto numa máquina Linux como disco local. Sugerido como alternativa a bucket para backup dos 380TB. |
| **Egress** | Tráfego de saída de dados de um provedor cloud. Todas as clouds cobram por egress — em caso de restore de 380TB, custo seria significativo. OCI cobra saída, não entrada (ingress). |
| **Certificate Manager (GCP)** | Serviço do Google Cloud para gerenciar certificados TLS. Integra com Let's Encrypt para gerar certificados automáticos, evitando problemas de CA não confiável. |
| **Let's Encrypt** | Autoridade certificadora gratuita e automatizada. Proposta de Wellington para certificados internos no GKE Standard, em vez da CA interna autoassinada do Sem Parar. |
| **Cert Manager** | Controller Kubernetes que automatiza emissão e renovação de certificados TLS (integra com Let's Encrypt, Certificate Manager, etc.). Já instalado no cluster GKE Standard por Léo. |
| **Gringo VPC** | Projeto GCP específico onde rodam os workloads produtivos da Gringo e que tem comunicação com ambiente PCI da Zapay. Escopo da auditoria PCI. |
| **Port-forward** | Comando `kubectl port-forward` que cria um túnel local para acessar serviços dentro do cluster Kubernetes sem expor externamente. Usado por Léo como alternativa para acessar ArgoCD. |
| **Samuel** | DBA responsável pela base Oracle de 380TB. Guilherme vai avaliar opções de contingência com ele. |
| **Feriado 10/07** | Quinta-feira é feriado. Alguns membros podem emendar a sexta-feira. |
